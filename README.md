# Tip & Payment App (Miyoshi)

「肉の匠 三芳」様のトーン＆マナーに合わせた、iPad向けのチップ計算・会計確認用Webアプリです。
完全無料で維持・運用できるよう、**GitHub Pages（フロントエンド）** と **Google Apps Script（バックエンド）** の組み合わせで構築されています。Node.jsやローカルサーバーは不要で、純粋なHTML・Vanilla JSとして動作します。

## ディレクトリ構成
```text
/
├── index.html       ... アプリ本体（エントリポイント）
├── styles/
│   ├── main.css     ... アプリのレイアウト・ボタン等のスタイル
│   └── theme.css    ... トンマナ（色、フォント、背景など）の定義
├── js/
│   ├── app.js       ... 画面遷移のルーティング処理
│   ├── store.js     ... アプリ全体の状態・金額データの管理機能
│   ├── screens/     ... 画面ごとのコンポーネント
│   │   ├── staffScreen.js    （スタッフ用入力画面）
│   │   ├── customerScreen.js （お客様用チップ選択画面）
│   │   └── thankYouScreen.js （完了・送信画面）
│   └── services/
│       └── sheetApi.js       （Googleスプレッドシートへの送信処理）
└── gas-script.js    ... Google Apps Scriptで使うバックエンド用コード
```

## 運用ステップ

本アプリを実際に稼働させるためには、以下の「フロントエンドの公開」と「バックエンドの連携」の2つを設定する必要があります。

### A. アプリ本体の公開（GitHub Pages）
1. 本プロジェクトのファイル一式をGitHubリポジトリにプッシュします。
2. GitHubのリポジトリ設定（Settings）から **Pages** を開きます。
3. `Source` を `Deploy from a branch` とし、ブランチを `main` (または `master`) にしてSaveします。
4. 数分待つとURLが発行されます。iPadのSafariでそのURLを開き、共有ボタンから**「ホーム画面に追加」**を選ぶと、ネイティブアプリのように全画面で動作します。

### B. 売上記録の連携設定（Google スプレッドシート）
アプリから選ばれたデータをスプレッドシートに記録するための手順です。

1. ご自身のGoogleアカウントで、新しく「Google スプレッドシート」を作成します。
2. メニューバーから **「拡張機能」** > **「Apps Script」** を開きます。
3. デフォルトで入力されているコードをすべて削除し、プロジェクト内の `gas-script.js` の中身をコピーして貼り付けます。
4. エディタ右上の **「デプロイ」** > **「新しいデプロイ」** を選びます。
5. 左の歯車アイコン ⚙️ をクリックし、**「ウェブアプリ」** を選択します。
6. 設定を以下の通りにします。
   - 実行するユーザー: `自分`
   - アクセスできるユーザー: `全員`
7. **「デプロイ」** をクリックし、アクセス承認が求められた場合は「詳細」から進んで許可します。
8. 発行された **「ウェブアプリのURL（https://script.google.com/...）」をコピー**します。
9. 本プロジェクトの `js/services/sheetApi.js` 内にある `GAS_WEB_APP_URL` 変数の中に、先ほどコピーしたURLを貼り付けて保存（コミット＆プッシュ）します。

## 開発・カスタマイズについて

- **カラー・フォントの変更**:
  色味やフォント・余白はすべて `styles/theme.css` の冒頭にあるVariables（変数群）で一括管理されています。
- **チップのパーセンテージの変更**:
  `js/screens/customerScreen.js` の上部 `tips` 配列の数値を書き換えるだけで、選択肢を自由に変更できます。
